## 📚 Knowledge: What “Best Documentation” Looks Like

Based on engineering playbooks from Google, Microsoft, CNCF, and industry best practices, 
**best technical documentation** is:

1. **Complete** – Covers system-level and component-level details (from overview to runbooks).  
2. **Consistent** – Uses the same structure across all repos, so developers know where to look.  
3. **Navigable** – Cross-links system docs and repo docs; diagrams aid understanding.  
4. **Visual** – Architecture, flows, dependencies, and schemas represented with diagrams (e.g., Mermaid).  
5. **Actionable** – Includes setup guides, troubleshooting, runbooks, and deployment steps.  
6. **Living** – Contains changelogs, TODOs, and roadmaps so it evolves with the system.  
7. **Audience-aware** – Speaks both to engineers (technical depth) and stakeholders (functional overview).
